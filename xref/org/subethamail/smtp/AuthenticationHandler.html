3   /**
4    * The interface that enables challenge-response communication necessary for
5    * SMTP AUTH.
6    * <p>
7    * Since the authentication process can be stateful, an instance of this class
8    * can be stateful too.<br>
9    * Do not share a single instance of this interface if you don't explicitly need
10   * to do so.
11   *
12   * @author Marco Trevisan &lt;mrctrevisan@yahoo.it&gt;
13   * @author Jeff Schnitzer
14   * @author Scott Hernandez
15   */
16  public interface AuthenticationHandler {
17  	/**
18  	 * Initially called using an input string in the RFC2554 form: "AUTH
19  	 * &lt;mechanism&gt; [initial-response]". <br>
20  	 * This method must return text which will be delivered to the client, or null
21  	 * if the exchange has been completed successfully. If a response is provided to
22  	 * the client, this continues the exchange - there will be another auth() call
23  	 * with the client's response.
24  	 * <p>
25  	 * Depending on the authentication mechanism, the handshaking process may
26  	 * require many request-response passes. This method will return
27  	 * <code>null</code> only when the authentication process is finished.
28  	 * <p>
29  	 * AuthenticationHandlers are associated with a single authentication exchange.
30  	 * If the exchange is stopped (i.e. fails) and is restarted, a new
31  	 * AuthenticationHandler is created. Upon successful authentication, your
32  	 * implementation of this object becomes part of the MessageContext. Your
33  	 * MessageHandler may upcast your AuthenticationHandler to obtain further
34  	 * information, such as identity.
35  	 * <p>
36  	 * AuthenticationHandlers do not need to handle the "*" cancel response; this is
37  	 * handled by the framework.
38  	 *
39  	 * @return <code>null</code> if the authentication process is finished,
40  	 *         otherwise a string to hand back to the client.
41  	 * @param clientInput The client's input, eg "AUTH PLAIN dGVzdAB0ZXN0ADEyMzQ="
42  	 * @throws org.subethamail.smtp.RejectException if authentication fails.
43  	 */
44  	String auth(String clientInput) throws RejectException;
45  
46  	/**
47  	 * If the authentication process was successful, this returns the identity of
48  	 * the user. The type defining the identity can vary depending on the
49  	 * authentication mechanism used, but typically this returns a String username.
50  	 * If authentication was not successful, the return value is undefined.
51  	 */
52  	Object getIdentity();
53  }
